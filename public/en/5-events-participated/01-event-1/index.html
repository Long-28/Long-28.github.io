AWS First Cloud AI Journey Community Day 2026

🎯 Context and participation objectives

Context

As AI and Cloud Computing continue to evolve rapidly, AWS First Cloud AI Journey Community Day 2026 connected Vietnam’s Cloud and AI community and shared real-world enterprise deployment practices on AWS.

Personal objectives

  • Learn about Generative AI, Multi-Agent Architecture, and Cloud infrastructure optimization.
  • Study how enterprises deploy large-scale AI/ML applications on AWS.
  • Apply Serverless, security, and performance best practices to GreenLens Kids.
  • Network with speakers, practitioners, and AWS User Group Vietnam.

Overview

This community event brought together AWS practitioners, speakers, and technology learners to discuss enterprise Cloud Computing, Artificial Intelligence, and modern AWS architectures.

Program highlights

  • Enterprise Multi-Agent Architecture: a credit-scoring solution using specialized agents, Bedrock Guardrails, Lambda, ECR, API Gateway, and AgentCore Runtime. The case reduced processing time from 2–3 weeks to 2–4 hours and reduced processing cost by about 95%.
  • LLM uncertainty: non-determinism caused by floating-point processing and inference batching, with techniques such as structured output, function calling, and majority voting to reduce variation.
  • Amazon CloudFront: HTTP/3, Gzip, Brotli, Origin Access Control, VPC origins, and AWS Shield for performance and security.
  • Amazon Q for enterprise: connecting business data sources and supporting Sales, Marketing, HR, and Customer Support workflows.
  • Context and the Second AI Brain: using goals, relevant information, constraints, success criteria, S3, vector databases, and Bedrock.
  • UTMorpho hackathon: a 36-hour AI project using S3, CloudFront, API Gateway, Lambda, and Bedrock.

Key takeaways

  • Enterprise AI is moving from single-agent designs toward specialized multi-agent systems.
  • Context quality matters more than simply giving an AI system more data.
  • LLM outputs should be treated as probabilistic and tested accordingly.
  • CloudFront is both a performance and security component, not only a CDN.

Reflection

This was my first AWS community event. The speakers showed how Bedrock, CloudFront, Lambda, API Gateway, S3, and ECR are combined in real systems. The Multi-Agent and UTMorpho sessions especially encouraged me to design AWS projects with scalability, security, and enterprise readiness in mind.

🏆 Achievements, evidence and personal contribution

Lessons learned

  • Understood Multi-Agent design thinking and data controls with AWS Bedrock Guardrails.
  • Learned how CloudFront, OAC, and AWS Shield improve application protection, performance, and cost control.
  • Learned to use the Context Framework to improve prompting and handle LLM uncertainty.

Evidence and personal contribution

  • Asked the speaker a question about OAC combined with an S3 Cross-Region Bucket.
  • Joined discussions, connected with the community, and shared the GreenLens Kids direction.
  • Prepared a knowledge summary and shared it with the internship group.

💡 Overall evaluation and reflection

The event narrowed the gap between theory and practice through cases from VPBank, Cloud Kinetics, and the AWS community. The lessons about flexible Serverless systems and safe Generative AI directly contributed ideas to the GreenLens Kids architecture.
