Overview#

The meetup connected students, software engineers, and AWS learners through practical talks about Cloud infrastructure, DevOps, security, AI, and career development.

Program highlights#

  • Docker: containers, images, registries, and the Build–Run–Deploy workflow in DevOps and Cloud.
  • AWS WAF and machine learning: detecting abnormal behavior and automatically blocking Web attacks.
  • Godot and AWS WebSocket: real-time synchronization and backend design for multiplayer games.
  • Effective teamwork: task allocation, communication, progress management, and practical project collaboration.
  • Amazon Neptune and GraphRAG: graph databases and knowledge bases combined with large language models.
  • From IT Helpdesk to Senior Sysadmin: self-learning Linux and AWS while transitioning into Cloud infrastructure.

Key takeaways#

  • Docker is an important foundation for modern application delivery.
  • WebSocket fits systems that require real-time communication.
  • Neptune and GraphRAG provide a useful approach to connected knowledge.
  • Cloud careers require continuous, hands-on learning and community participation.

Reflection#

The meetup broadened my understanding from infrastructure and security to game systems, graph databases, and career development. It gave me practical examples of how Docker, AWS WAF, Neptune, and WebSocket are used, while the career talk provided realistic motivation for continuing my AWS and Linux learning path.

Lessons learned#

  • Learned application packaging with Docker and real-time communication through AWS WebSocket API.
  • Understood how GraphRAG on Amazon Neptune enriches AI context.
  • Learned Git Flow teamwork practices and a structured Linux/Cloud learning path.

Evidence and personal contribution#

  • Actively discussed long-term AWS WAF Logs storage cost reduction with the speaker.

💡 Overall evaluation and reflection#

The meetup provided practical perspectives on infrastructure, security, GraphRAG, WebSocket, and career development. Vinh Tran’s talk strengthened my commitment to Cloud and to maintaining a consistent self-learning habit.
